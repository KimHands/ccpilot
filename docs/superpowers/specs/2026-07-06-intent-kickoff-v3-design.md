# v0.3.0 — 의도 기반 킥오프 (intent-assisted kickoff) 설계 문서

- 작성일: 2026-07-06
- 상태: 설계 확정 (코덱스 리뷰 "build with changes" 반영)
- 선행: A(프리셋·단계) · B(에이전트 라우팅) · v2(SessionStart 훅) — 완료·설치됨

## 1. 목적

프로젝트 시작 발화("이 프로젝트 초기화해줘" / "프리셋 정해줘")를 감지하면, **프리셋을 자동 추론 → 프리뷰 → 확인 → `/project-init` 실행**까지 이어준다. 프리셋 선택·타이핑 부담을 없애되, **파일 쓰기 전 반드시 확인**한다.

## 2. 코덱스 반영 원칙 (핵심)

- **확률적 트리거로 파일 자동 작성 금지** → 쓰기 전 확인 게이트 필수(코덱스 #1).
- **재시작 최소화 최적화 폐기** → init은 denylist로 비프리셋을 끄기까지 하므로 거의 항상 설정이 바뀜 → "init 후 1회 재시작"으로 단순·정확(코덱스 #2).
- **intent 추론은 모델(스킬 본문)이**, 결정적 코드는 검증·언어감지·프리뷰만(코덱스 #5). 키워드 매칭을 authoritative로 쓰지 않는다.
- **트리거는 좁게** — 명시적 초기화/셋업 발화만. 일반 "X 만들거야"엔 안 걸린다(코덱스 #4).
- **낮은 확신 → 자동 실행 X, 명령 추천만**(코덱스 #9).
- **프리뷰·되돌리기** — 기존 `/project-init X --dry-run` 재사용(코덱스 #6).

## 3. 비목표

- **무확인·무재시작 조용한 완전자동** — 위험·아키텍처상 불가. 확인 게이트 유지.
- 새 결정적 추론 엔진 — 모델이 추론, 신규 파이썬 최소.
- 세션 중 플러그인 hot-toggle — 여전히 세션 시작 로드(재시작 필요).

## 4. 아키텍처

기존 자산 재사용이 핵심 — 신규 코드 최소.

```
~/.claude/skills/project-kickoff/SKILL.md   # 신규: 자동 트리거 스킬(모델 주도)
install.sh                                    # 확장: 스킬 복사
docs/…                                        # spec + plan
```

- **project-kickoff 스킬**: 자동 트리거(좁은 description). 본문이 Claude에게 아래 흐름을 지시.
- **재사용**: 언어 감지·프리뷰는 이미 있는 `python3 ~/.claude/pw/cli.py init <preset> --dry-run`(enabled/disabled/missing JSON 반환)로 처리. 신규 파이썬 없음.

## 5. 스킬 흐름 (SKILL.md 본문)

1. **트리거 판별**: "이 폴더/프로젝트를 초기화/셋업", "Claude Code 프리셋 정해줘", "프로젝트 워크플로우 설정" 등 **명시적 킥오프 의도**일 때만. 단순 "X 만들거야"·브레인스토밍·리팩터링 언급엔 발동하지 않는다.
2. **프리셋 추론(모델)**: 사용자 메시지 + 폴더 파일(package.json/pyproject.toml 등)로 5개 프리셋 중 후보 판단.
   - 참고 힌트(authoritative 아님): FastAPI·Django·API·백엔드→backend-api · React·Next·UI→web-fullstack · 보안·감사→security-audit · 슬라이드·PPT·문서→docs-deck · 그 외→minimal.
3. **프리뷰**: `python3 ~/.claude/pw/cli.py init <preset> --dry-run` 실행 → 활성/비활성 플러그인·재시작 필요 표시.
4. **확인 게이트**: "이 폴더를 `<preset>`로 초기화할까요? (활성 N · 비활성 M · 재시작 1회)". **승인 전 파일 쓰기 금지.**
5. **실행**: 승인 시 `python3 ~/.claude/pw/cli.py init <preset>` → "새 세션에서 로드됨(재시작 1회)" 안내.
6. **낮은 확신/모호**: 자동 실행하지 않고, 후보 프리셋과 정확한 명령(`/project-init <preset>`)만 추천. 폴더 초기화 자체가 모호하면 **행동 동의부터** 묻는다("이 폴더를 초기화할까요?").
7. **기존 프로젝트 보호**: 이미 `.claude/project-state.json`이 있으면 재초기화 대신 현황(`/project-status`)을 안내.

## 6. 안전·되돌리기

- 파일 쓰기는 승인 후에만. init은 비파괴(기존 설정·CLAUDE.md 마커 밖 보존, `--force` 없이는 재초기화 거부).
- 프리뷰(`--dry-run`)로 변경 대상을 먼저 보여준다.

## 7. 테스트 전략

스킬은 마크다운(모델 행동)이라 필수 요소 존재를 검증:
- 좁은 트리거 문구 · dry-run 프리뷰 단계 · **확인 게이트 문구** · 낮은 확신 시 추천만 · 재시작 안내 · 기존 프로젝트 보호 언급.
- 오트리거 방지: "단순 'X 만들거야'엔 발동 안 함" 문구 존재.
- (추론 정확도는 모델 영역 — 자동 단위테스트 대상 아님.)
